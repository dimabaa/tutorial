# 🐧 Базовые команды Linux: краткий практический справочник для начинающих

> Если вы только начинаете работать с Linux, терминал может показаться сложным. На самом деле достаточно освоить несколько десятков базовых команд, чтобы уверенно управлять системой, файлами и процессами.

---

# 📂 Навигация по файловой системе

## Узнать текущую директорию

```bash
pwd
```

Вывод:

```text
/home/user/projects
```

Команда показывает, где вы сейчас находитесь.

---

## Посмотреть содержимое каталога

```bash
ls
```

Пример:

```text
documents
downloads
script.sh
```

### Полезные варианты

Показать подробную информацию:

```bash
ls -l
```

Показать скрытые файлы:

```bash
ls -a
```

Показать всё вместе:

```bash
ls -la
```

---

## Перейти в другую папку

```bash
cd documents
```

Вернуться на уровень выше:

```bash
cd ..
```

Перейти в домашнюю директорию:

```bash
cd ~
```

Перейти в корень системы:

```bash
cd /
```

---

# 📄 Работа с файлами

## Создать пустой файл

```bash
touch notes.txt
```

Проверяем:

```bash
ls
```

Результат:

```text
notes.txt
```

---

## Просмотреть содержимое файла

```bash
cat notes.txt
```

Пример вывода:

```text
Hello Linux!
```

---

## Просматривать длинные файлы постранично

```bash
less logfile.log
```

Навигация:

- `Space` — следующая страница
- `b` — предыдущая
- `q` — выход

---

## Первые строки файла

```bash
head file.txt
```

Первые 5 строк:

```bash
head -n 5 file.txt
```

---

## Последние строки файла

```bash
tail file.txt
```

Следить за логом в реальном времени:

```bash
tail -f app.log
```

Очень полезно для мониторинга приложений.

---

# 📁 Создание и удаление директорий

## Создать папку

```bash
mkdir project
```

Создать вложенные папки:

```bash
mkdir -p project/src/api
```

---

## Удалить пустую папку

```bash
rmdir project
```

---

## Удалить папку со всем содержимым

```bash
rm -r project
```

Принудительное удаление:

```bash
rm -rf project
```

⚠️ Будьте осторожны: команда удаляет файлы без возможности восстановления.

---

# 📋 Копирование и перемещение

## Копировать файл

```bash
cp file.txt backup.txt
```

---

## Копировать директорию

```bash
cp -r project project_backup
```

---

## Переместить файл

```bash
mv file.txt documents/
```

---

## Переименовать файл

```bash
mv old.txt new.txt
```

---

# 🔍 Поиск файлов

## Найти файл по имени

```bash
find . -name "config.yaml"
```

Пример вывода:

```text
./app/config.yaml
```

---

## Поиск с маской

```bash
find . -name "*.log"
```

Найдёт все лог-файлы.

---

# 🔎 Поиск текста

Предположим, есть файл:

```text
server started
database connected
error connection refused
```

Найдём строку с ошибкой:

```bash
grep error app.log
```

Результат:

```text
error connection refused
```

---

## Игнорировать регистр

```bash
grep -i error app.log
```

Найдёт:

```text
ERROR
Error
error
```

# 👤 Работа с пользователями

## Узнать текущего пользователя

```bash
whoami
```

Пример:

```text
ubuntu
```

---

## Информация о пользователе

```bash
id
```

Результат:

```text
uid=1000(user) gid=1000(user)
```

---

# ⚙️ Управление процессами

## Посмотреть процессы

```bash
ps aux
```

---

## Найти процесс

```bash
ps aux | grep nginx
```

---

## Показать процессы в реальном времени

```bash
top
```

или более удобный вариант:

```bash
htop
```

> `htop` может потребовать установки.

---

## Завершить процесс

Сначала узнаём PID:

```bash
ps aux | grep nginx
```

Например:

```text
root  1234 nginx
```

Останавливаем:

```bash
kill 1234
```

Принудительно:

```bash
kill -9 1234
```

---

# 🌐 Сетевые команды

## Проверка доступности сервера

```bash
ping google.com
```

Пример:

```text
64 bytes from ...
```

Остановить:

```text
Ctrl + C
```

---

## Проверка открытых портов

```bash
ss -tulpn
```

---

## Узнать IP-адрес

```bash
ip a
```

или

```bash
hostname -I
```

---

# 📦 Работа с пакетами

## Ubuntu / Debian

Обновить список пакетов:

```bash
sudo apt update
```

Обновить систему:

```bash
sudo apt upgrade
```

Установить программу:

```bash
sudo apt install nginx
```

Удалить программу:

```bash
sudo apt remove nginx
```

---

# �� Права доступа

## Посмотреть права

```bash
ls -l
```

Пример:

```text
-rwxr-xr-- script.sh
```

Расшифровка:

| Символ | Значение |
|----------|----------|
| r | чтение |
| w | запись |
| x | выполнение |

---

## Сделать файл исполняемым

```bash
chmod +x script.sh
```

Запуск:

```bash
./script.sh
```

---

## Изменить владельца

```bash
sudo chown user:user file.txt
```

---

## Размер папок

```bash
du -sh *
```

```bash

```


# 🚀 Полезные комбинации

| Команда | Назначение |
|----------|------------|
| `history` | История команд |
| `clear` | Очистить терминал |
| `man ls` | Документация по команде |
| `Ctrl + C` | Остановить процесс |
| `Ctrl + L` | Очистить экран |
| `Tab` | Автодополнение |
| `↑ ↓` | История команд |

# 🎯 Мини-практика

Создадим проект с нуля:

```bash
mkdir linux-demo
cd linux-demo

touch app.py
mkdir logs

echo "Hello Linux" > app.py

cat app.py

ls -la
```

Результат:

```text
linux-demo/
├── app.py
└── logs/
```

# Заключение

После освоения этих команд вы сможете:

- уверенно перемещаться по файловой системе;
- создавать, копировать и удалять файлы;
- искать данные и анализировать логи;
- управлять процессами;
- работать с сетью;
- устанавливать программное обеспечение;
- понимать права доступа в Linux.

Для большинства повседневных задач системного администратора, DevOps-инженера или разработчика именно эти команды используются ежедневно.
